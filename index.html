NAEP Style Guide — UX Design System eNAEP platform · Educational Testing Service · live artifact

A living design system governing UI presentation across all student-facing interfaces for the National Assessment of Educational Progress — one of the largest standardized assessment programs in the United States. This is documentation architecture at national scale: specifying how every UI element behaves, reads, and adapts across discrete items, scenario-based tasks, and interactive components for millions of test takers.

The guide covers eight primary domains — Basics (color, typography, keyboard navigation), UI Elements (buttons, dialogs, controls), Features & Tools (calculators, text-to-speech), Interactive Stimuli (rulers, sliders, sortable tables), Response Types, Instruction, and Environments. Each component includes live demos, interactive examples, and accessibility specifications.

Shown below: the Transformable Objects section — design specs for interactive components that test takers can resize, rotate, and translate, with precise handle states, cursor behaviors, and accessibility modes documented for engineering implementation.

Research Background
jconor@portfolio:~$ cat research-background/masters-educational-psychology.md
M.S. · Educational Psychology 2022 · UX & accessibility in digital assessment

My master's thesis investigated how UX design and accessibility principles could be applied to improve the experience of digital assessment platforms — the high-stakes, standardized testing environments used by millions of K–12 students.

The research focused on the intersection of cognitive load theory, accessible interface design, and assessment validity — specifically how poor UX can introduce construct-irrelevant variance, meaning students underperform not because of what's being measured, but because the interface itself creates barriers. My work proposed design frameworks for reducing that friction across input modalities, navigation patterns, and adaptive content presentation.

This research directly informs my current practice: the same principles that govern accessible assessment design — clarity, reduced cognitive overhead, predictable interaction patterns — apply directly to the design of AI-facing documentation systems and agent UX.
